# Setting Kumon dengan github private
Berikut ini rangkaian setting nya agar server bisa berkomunikasi dengan git hub yang private menggunakan teknologi terbaru.

## Generate SSH Key untuk Server (Ed25519 - Recommended)

Gunakan Ed25519 (lebih modern, aman, dan efisien dibanding RSA):

```bash
ssh-keygen -t ed25519 -C "server@domain.com" -f ~/.ssh/id_ed25519
```

**Alternatif dengan RSA 4096 (jika server tidak mendukung Ed25519):**
```bash
ssh-keygen -t rsa -b 4096 -C "server@domain.com" -f ~/.ssh/id_rsa
```

Saat diminta passphrase, bisa dikosongkan atau diberi passphrase untuk keamanan lebih (recommended untuk production).

## Tambahkan SSH Key ke GitHub

### Lihat Public Key
```bash
cat ~/.ssh/id_ed25519.pub
```

### Langkah-langkah di GitHub:

1. Buka [GitHub Settings](https://github.com/settings/keys)
2. Pilih **SSH and GPG keys** di sidebar
3. Klik **New SSH Key**
4. Masukkan nama key (misalnya: `Server Global`)
5. Pilih **Key type**: `Authentication Key`
6. Paste isi dari `id_ed25519.pub` ke kolom **Key**
7. Klik **Add SSH Key**

### (Opsional) Tambahkan Signing Key untuk Commit Signature

Untuk security yang lebih baik, GitHub merekomendasikan signing key terpisah:

```bash
ssh-keygen -t ed25519 -C "server@domain.com" -f ~/.ssh/id_ed25519_signing
```

Kemudian ulangi langkah di GitHub dengan memilih **Key type**: `Signing Key`

## Uji Coba Koneksi ke GitHub

```bash
ssh -T git@github.com
```

Jika berhasil, akan menampilkan:
```
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

## Konfigurasi SSH Secara Global untuk GitHub

```bash
nano ~/.ssh/config
```

Isi dengan konfigurasi berikut:

```
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519
    IdentityFile ~/.ssh/id_ed25519_signing
    AddKeysToAgent yes
    StrictHostKeyChecking accept-new
    IdentitiesOnly yes
```

**Penjelasan:**
- `IdentityFile` (multiple): Mendukung authentication dan signing key
- `AddKeysToAgent yes`: Menyimpan key di SSH agent agar tidak perlu passphrase setiap kali
- `StrictHostKeyChecking accept-new`: Lebih aman dari `no`, hanya accept host key baru sekali saja
- `IdentitiesOnly yes`: Hanya gunakan key yang sudah didefinisikan

Simpan dengan `Ctrl + X` → `Y` → `Enter`

## Setup SSH Agent (Optional tapi Recommended)

Jika menggunakan passphrase, setup SSH agent agar passphrase hanya diminta sekali:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

Untuk otomatis saat startup, tambahkan ke `~/.bashrc` atau `~/.zshrc`:

```bash
if [ -z "$SSH_AUTH_SOCK" ]; then
    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
fi
```

## Clone Repository Private

### Full Repository
```bash
git clone git@github.com:username/repo-private.git
```

### Branch Tertentu
```bash
git clone --branch NAMA_BRANCH --single-branch git@github.com:username/repo-private.git
```

### Dengan Depth (untuk repo besar)
```bash
git clone --depth 1 --branch NAMA_BRANCH git@github.com:username/repo-private.git
```

## Troubleshooting

### Permission denied (publickey)
- Pastikan SSH key sudah ditambahkan ke GitHub
- Cek permissions: `chmod 600 ~/.ssh/id_ed25519` dan `chmod 644 ~/.ssh/id_ed25519.pub`
- Test koneksi: `ssh -T git@github.com`

### SSH Key tidak ter-load
```bash
# Cek apakah key ter-load
ssh-add -L

# Load ulang jika belum
ssh-add ~/.ssh/id_ed25519
```

### Masalah Strict Host Key
Jika ada warning tentang host key, jalankan:
```bash
ssh-keyscan -H github.com >> ~/.ssh/known_hosts
```

## Referensi & Best Practices

- [GitHub SSH Documentation](https://docs.github.com/en/authentication/connecting-to-github-with-ssh)
- Gunakan Ed25519 untuk produksi modern
- Jangan pernah share private key
- Gunakan passphrase untuk ekstra keamanan
- Rotate key secara berkala (setiap tahun atau sesuai policy)
- Gunakan signing key terpisah untuk commit verification
