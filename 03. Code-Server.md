## 1. Prasyarat

Sebelum memulai, pastikan server kamu memenuhi spesifikasi berikut:

| Komponen | Minimum |
|----------|---------|
| OS | Ubuntu 20.04 / 22.04 / 24.04 LTS |
| RAM | 1 GB (disarankan 2 GB+) |
| CPU | 1 vCPU |
| Storage | 10 GB kosong |
| Akses | Root / sudo |

## 2. Instalasi

### 2.1 Update Sistem & Install Dependensi

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget unzip
```

> **Mengapa perlu update?** Memastikan semua paket sistem terbaru sebelum instalasi untuk menghindari konflik dependensi.

### 2.2 Install Code Server

Gunakan skrip instalasi resmi dari cdr.sh:

```bash
curl -fsSL https://code-server.dev/install.sh | sh
```

**Penjelasan flag curl:**
- `-f` → Gagal diam-diam jika ada error HTTP
- `-s` → Mode silent (tanpa progress bar)
- `-S` → Tampilkan error meski mode silent
- `-L` → Ikuti redirect otomatis

### 2.3 Verifikasi Instalasi

```bash
code-server --version
```

Output yang diharapkan:
```
4.x.x xxxxxxx linux-amd64
```

## 3. Konfigurasi Dasar

### 3.1 Buat / Edit File Konfigurasi

File konfigurasi terletak di `~/.config/code-server/config.yaml`. Edit menggunakan nano:

```bash
nano ~/.config/code-server/config.yaml
```

### 3.2 Isi Konfigurasi

```yaml
bind-addr: 0.0.0.0:8080
auth: password
password: GantiDenganPasswordKuat123!
cert: false
```

**Penjelasan setiap parameter:**

| Parameter | Nilai | Keterangan |
|-----------|-------|------------|
| `bind-addr` | `0.0.0.0:8080` | Dengarkan di semua interface, port 8080 |
| `auth` | `password` | Wajib login dengan password |
| `password` | *(isi sendiri)* | Password untuk masuk ke Code Server |
| `cert` | `false` | Nonaktifkan SSL bawaan (gunakan Nginx/Caddy untuk SSL) |

> ⚠️ **Penting:** Jangan gunakan password yang lemah. Gunakan kombinasi huruf besar, kecil, angka, dan simbol minimal 12 karakter.

### 3.3 Simpan & Keluar

Tekan `Ctrl+O` → Enter → `Ctrl+X` untuk menyimpan dan keluar dari nano.

## 5. Konfigurasi Systemd Service

Agar Code Server berjalan otomatis di background dan tetap aktif setelah reboot, konfigurasi sebagai systemd service.

### 5.1 Buat File Service

```bash
sudo nano /etc/systemd/system/code-server.service
```

### 5.2 Isi File Service

```ini
[Unit]
Description=Code Server
After=network.target

[Service]
Type=simple
User=ubuntu
ExecStart=/usr/bin/code-server --bind-addr 0.0.0.0:8080
Restart=always
RestartSec=5
Environment=HOME=/home/ubuntu

[Install]
WantedBy=multi-user.target
```

**Penjelasan setiap bagian:**

**[Unit]**
- `Description` → Nama deskriptif service
- `After=network.target` → Tunggu network aktif sebelum service berjalan

**[Service]**
- `Type=simple` → Proses utama langsung dijalankan
- `User=ubuntu` → Jalankan sebagai user `ubuntu` (bukan root!)
- `ExecStart` → Perintah untuk memulai Code Server
- `Restart=always` → Restart otomatis jika service mati
- `RestartSec=5` → Tunggu 5 detik sebelum restart
- `Environment=HOME` → Set home directory agar config terbaca dengan benar

**[Install]**
- `WantedBy=multi-user.target` → Aktifkan saat sistem masuk mode multi-user

> ⚠️ **Sesuaikan `User`** dengan username server kamu. Cek dengan perintah `whoami`.

### 5.3 Aktifkan & Jalankan Service

```bash
# Reload konfigurasi systemd
sudo systemctl daemon-reload

# Aktifkan agar otomatis berjalan saat boot
sudo systemctl enable code-server

# Jalankan sekarang
sudo systemctl start code-server

# Cek status
sudo systemctl status code-server
```

### 5.4 Output Status yang Normal

```
● code-server.service - Code Server
     Loaded: loaded (/etc/systemd/system/code-server.service; enabled; vendor preset: enabled)
     Active: active (running) since ...
   Main PID: XXXX (node)
      Tasks: XX
     Memory: XX.XM
        CPU: XXXms
```

### 5.5 Perintah Manajemen Service

```bash
sudo systemctl stop code-server      # Hentikan service
sudo systemctl restart code-server   # Restart service
sudo systemctl disable code-server   # Nonaktifkan dari autostart
journalctl -u code-server -f         # Lihat log realtime
```

## 6. Keamanan & Hardening

### 6.1 Firewall dengan UFW

Jangan biarkan port 8080 terbuka ke publik. Batasi akses hanya dari IP tertentu:

```bash
# Install UFW jika belum ada
sudo apt install -y ufw

# Izinkan SSH agar tidak terkunci
sudo ufw allow 22/tcp

# Izinkan akses Code Server hanya dari IP spesifik
# Ganti 203.0.113.10 dengan IP publik kamu
sudo ufw allow from 203.0.113.10 to any port 8080

# Aktifkan firewall
sudo ufw enable

# Cek status
sudo ufw status verbose
```

> 🔐 **Jangan buka port 8080 ke semua IP (0.0.0.0/0)** — ini akan membuat Code Server kamu terbuka ke seluruh internet.

### 6.2 Reverse Proxy dengan Nginx + HTTPS (Direkomendasikan)

Gunakan Nginx sebagai reverse proxy dengan SSL agar koneksi terenkripsi.

**Install Nginx & Certbot:**

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

**Buat konfigurasi Nginx:**

```bash
sudo nano /etc/nginx/sites-available/code-server
```

**Isi konfigurasi:**

```nginx
server {
    listen 80;
    server_name kode.domainmu.com;

    location / {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection upgrade;
        proxy_set_header Accept-Encoding gzip;
        proxy_http_version 1.1;
    }
}
```

**Aktifkan konfigurasi:**

```bash
sudo ln -s /etc/nginx/sites-available/code-server /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

**Pasang SSL dengan Certbot:**

```bash
sudo certbot --nginx -d kode.domainmu.com
```

Setelah ini Code Server dapat diakses via `https://kode.domainmu.com` dengan HTTPS otomatis.

### 6.3 Ubah Password Secara Berkala

Edit file konfigurasi dan restart service:

```bash
nano ~/.config/code-server/config.yaml
# Ubah nilai `password`

sudo systemctl restart code-server
```

### 6.4 Gunakan SSH Tunneling (Alternatif Paling Aman)

Jika tidak ingin expose port sama sekali, gunakan SSH tunnel dari komputer lokal:

```bash
# Jalankan di komputer lokal (bukan server)
ssh -N -L 8080:localhost:8080 ubuntu@IP_SERVER_KAMU
```

Kemudian buka `http://localhost:8080` di browser lokal kamu. Tidak ada port yang perlu dibuka di firewall server.

### 6.5 Batasi Akses dengan Fail2Ban

Lindungi dari brute-force login:

```bash
sudo apt install -y fail2ban
sudo systemctl enable --now fail2ban
```

### 6.6 Checklist Keamanan

- [ ] Password kuat (minimal 12 karakter, kombinasi huruf+angka+simbol)
- [ ] Firewall aktif (UFW) — port 8080 tidak terbuka ke publik
- [ ] HTTPS aktif via Nginx + Certbot (jika expose ke internet)
- [ ] SSH tunnel digunakan jika hanya untuk akses pribadi
- [ ] Fail2Ban aktif
- [ ] Update sistem rutin: `sudo apt update && sudo apt upgrade -y`
- [ ] Code Server tidak berjalan sebagai root

## 8. Troubleshooting

### Service tidak mau start

```bash
# Cek log detail
journalctl -u code-server -n 50 --no-pager

# Pastikan path binary benar
which code-server
```

### Port 8080 sudah dipakai

```bash
# Cek proses yang menggunakan port 8080
sudo ss -tulnp | grep 8080

# Ganti port di config.yaml dan service file jika perlu
```

### Lupa password

```bash
nano ~/.config/code-server/config.yaml
# Edit baris `password:`
sudo systemctl restart code-server
```

### Tidak bisa akses dari browser

```bash
# Cek apakah service berjalan
sudo systemctl status code-server

# Cek firewall
sudo ufw status

# Cek apakah port terbuka
sudo ss -tulnp | grep 8080
```

### Permission denied saat start service

Pastikan `User=` di file service sesuai dengan user yang punya direktori home dan instalasi code-server:

```bash
whoami              # Cek username aktif
ls ~/.config/code-server/  # Pastikan config ada
```
